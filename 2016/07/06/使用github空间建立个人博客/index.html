使用github空间建立个人博客

很早之前就搭了个博客，放了些自己写的东西在上面。当时是申请的阿里云的服务器，最便宜的那种，六百多一年（实在没人方问）。后来停用的原因也很简单，工信部是备了案，需要在公安局再备一次，填写了几次表格都不太合标准，然后就把服务器关了。想一想自己实在挫，否则也不至于这么点事也搞不定。

后来朋友告诉我，github的空间搭博客不错，有300MB的空间，弄个静态的博客上去，完全胜任。我就动手试了试，果然可行，速度也过的去，反正也没人看，权当自己写日记了。

这里写一下搭建的过程，以及遇到过的坑。

域名么，早就有了，就是这个Beeper2G.com, 那么接下来在github上申请帐号吧。这里就遇到了一个问题，用的163邮箱，结果收不到github的认证邮件。查了查，把github.com放在白名单中，再重新生成认证邮件，然后就收到了…

接下来的事就很简单了，新建一个名子和帐户一样的工程，我这是里就是beeper2g.github.io。setting中可以自动生成博客模版，下一步下一步，然后就成功了。输入http://beeper2g.github.io试一下，能成功打开网站了。

工程根目录下有一个CNAME，里面放的就是域名，这个文件不能丢，丢了就解析不过来了。可以手动编辑这个文件，也可以在要setting中设置，设置成功后这个文件自动就修改成功了。修改完毕后，把你的域名解过来，带wwww的和不带的，都解析过来。再输入你的个人域名试试，应该可以打开了，打不开的话刷下dns看看解析有没问题。

使用hexo写博客。

自动生成的博客，是可以使用的，只是编辑发布有点麻烦，大家可以用官方的工具来做这些事情。我这里介绍一下hexo写博客。

我知道hexo是自己从网上自己搜的，开始以为这东西是基于express的网站容器，因为按hexo官方的介绍，生成网站后，hexo server就可以在本地看效果。然后我就在centos上搭了一个hexo环境。当我查怎么样使用forever让这个玩意跑起来的时候，发现这个东西只是不是生成静态网站的工具罢了。

hexo不多说，用起来很简单，有中文文档，这里说说我遇到的坑

  • 我用工具生成项目文件后，把除了node_modules以外的文件提交到了svn，然后另外一台机器上运行的npm install。后来发现运行hexo generate时，生成的网页文件不全。对比了一下，hexo init生成的文件中有package.json以外的包。

  • github是用用户名验证的，在另一台电脑提交的时候，出现了一个问题，其实我现在也没解决掉，只是绕过了这个问题：

    FATAL bash: /dev/tty: No such device or address
    error: failed to execute prompt script (exit code 1)
    fatal: could not read Username for ‘https://github.com‘: Invalid argument
    查了很多资料都不行，最后改了一下_config.yml，把https改成了git

    deploy:

    type: git
repository: git@github.com:lhfiii/lhfiii.github.io.git
branch: master

    然后生成key，用ssh方式提交，就成功了…

关于图片

图片最好不要往空间里丢，一是空间本身不大，二是图片资料访问太慢。这里建议大家用七牛的服务，用了一下，感觉不错。也可以用cloudinary的服务。这个就看个人喜好了，cloudinary功能也挺强大。
